Typedeffh_frame_readout_t

Frame readout are opaque objects. To read frame the user should initialize one using fh_frame_readout_create, and read over an opened fh_context_t using fh_context_read. It is a good practice to reuse for each read the same fh_frame_readout_t, and not to create a new one for each fh_context_read calls.

To clean up allocated ressources the user would need to use the fh_frame_readout_destroy function.

timeval * res )

Due to realtime issue implied by jitter and clock synchronization, this field could present jitter and irregularities (in some extreme cases, date will not even be monotonous in time). It should only be used to find a particular event in time. To measure time between frame, use the monotonous fh_frame_readout_timestamp function.
